# 0003. AI_MERGE as authoritative control plane

- Status: accepted
- Date: 2026-06-07

## Context

Managing a repository merge across multiple AI agent sessions and human contributors requires a durable, auditable, and version-controlled source of truth for the merge state and intended tasks.

## Decision

We designate the **`AI_MERGE/`** directory and the **`.idd/MANIFEST.tsv`** as the authoritative control plane for all integration operations. 
- `AI_MERGE/` contains the feature matrix, inventory reports, and the task backlog.
- `.idd/MANIFEST.tsv` provides a deterministic registry of all artifacts generated by the system.
- No external databases or hidden state stores are used.

## Consequences

- **Positive**: Full auditability via Git history; zero-setup "cold start" for fresh agents; clear separation between product code and integration metadata.
- **Neutral**: Requires discipline in committing `AI_MERGE/` updates alongside code changes.
- **Negative**: Adds non-product boilerplate to the repository root.
